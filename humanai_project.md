5. 第四轮回复（MNN 方案落地）

我给出基于 MNN 的完整实现路径：

5.1 模型获取

· 下载官方提供的 LFM2.5-1.2B-Thinking-MNN 系列模型（.mnn 格式，已 4-bit 量化）。
· 来源：Hugging Face 或 ModelScope。

5.2 环境编译（Android / iOS）

· Android：在 project/android 目录下编译，启用 LLM 和 OpenCL 支持：
  ```bash
  ../build_64.sh -DMNN_BUILD_LLM=true -DMNN_OPENCL=true -DMNN_USE_LOGCAT=true
  ```
· iOS：参考官方脚本。
· 可选优化：开启低内存模式和权重反量化 GEMM。

5.3 核心代码框架（C++）

· 加载模型 -> 创建 Session。
· 主循环（永不退出）：
  1. 检查是否有新用户输入，有则 token 化后注入输入张量，调用 runSession 处理。
  2. 执行一步自回归推理（net->runSession(session)）。
  3. 获取最新生成的 token。
  4. 检查是否为函数调用标记（如 <function_call>），若是则解析并执行本地函数（打印、硬件操作等），同时将函数结果作为新 token 放回，让模型继续思考。
  5. 检查是否为停止思考标记（可设计为永远不触发）。
· 关键：需妥善管理 KV Cache 以维持对话状态。

5.4 Python 备选（pymnn）

· 可用于快速验证，但移动端生产环境建议 C++。
